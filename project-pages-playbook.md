# Project pages playbook

How to turn a live project URL into a filled case-study page on this site.
Follow this end-to-end. Write the full case-study copy from the brief, live site, and
available project material. **Do not invent numbers, metrics, dates, or monetary
amounts.** Leave only those missing quantitative details as visible `[BRACKET]`
placeholders for Panagiotis to complete.

Reference build: the Sfera 102.2 page (`slug: "sfera-radio-website"`), done from the
single URL `https://sfera.gr`.

---

## What "filling a page" means

Two separate jobs, from different sources:

1. **Images** — real cover + gallery screenshots (from the live site via Playwright).
2. **Copy** — Problem / Context / Challenge / Process / Output / Result. Write these
   sections in full from the facts available in the brief and the live site. Do not
   invent numbers, metrics, dates, or monetary amounts.

Everything lives in one data file: [src/data/caseStudies.ts](src/data/caseStudies.ts).
Adding a project is editing that array — the page at `/work/<slug>/` is generated by
[src/pages/work/[slug].astro](src/pages/work/%5Bslug%5D.astro).

---

## Copy rules for agents

Before writing or editing any public-facing copy, read
[counter-ai-ism-guide.md](counter-ai-ism-guide.md). Treat it as a required writing
standard, not optional advice.

Use the facts from the brief, the live site, and material supplied by Panagiotis. Do
not turn inference into fact, add a result that has not been evidenced, or fill gaps
with generic marketing language. Complete every copy field. Use visible `[BRACKET]`
placeholders only for an unverified number, metric, date, or monetary amount.

### Write directly

- Lead with the project, user task, constraint, decision, or outcome itself.
- Name the actual people, services, pages, content, and actions involved.
- Prefer concrete nouns and verbs. Explain what changed or what people can do.
- Match certainty to the evidence: use “according to the brief”, “appears”, “may”, or
  “suggests” when the source does not establish a fact.
- Give each paragraph one job. Do not restate the point as a conclusion.

### Avoid generated-sounding copy

- Apply the central test: if a sentence could fit hundreds of unrelated projects after
  changing one or two nouns, rewrite it.
- Do not use broad scene-setting, fake urgency, simulated warmth, praise, or a generic
  closing offer.
- Avoid inflated or empty claims such as “seamless”, “intuitive”, “impactful”,
  “transformative”, “world-class”, “future-ready”, “robust”, “strategic”,
  “cohesive”, or “user-friendly” unless the copy immediately states the evidence.
- Avoid corporate abstractions such as “leverage”, “enable”, “empower”, “unlock”,
  “elevate”, “drive”, “align”, and “deliver value”. State the action instead.
- Avoid stock constructions such as “in today’s…”, “more than just…”, “not only X but
  also Y”, “whether you are…”, “at its core”, “ultimately”, “the key is”, “this
  ensures”, and “this allows”.
- Do not force groups of three, symmetrical sentences, adjective stacks, or a fixed
  Challenge / Process / Output / Result rhythm when the project does not support it.
- NEVER use em-dashes, at all costs. 
- In general avoid AI-isms.

### Punctuation and structure

- Do not use em dashes in public copy. Use a full stop, comma, colon, or a rewritten
  sentence.
- Vary sentence length and opening structure. Do not repeatedly start sentences with
  “This”, “It”, “By”, “With”, or “The site”.
- Use headings and lists only when they help people scan information.
- Use calls to action that name the action, such as “View project details” or “Send a
  project outline”, rather than vague prompts such as “Learn more” or “Get started”.

### Case-study standard

- `summary` identifies the project and the work done in plain language.
- `problem` states the actual constraint or user need, not a vague business benefit.
- Context, process, and output sections describe what is known. Write a qualitative
  result from the delivered work where appropriate, but do not claim a measurable
  result without a source.
- Keep project-specific terminology where it helps accuracy, but explain unfamiliar
  terms through the surrounding sentence.

## Content model (the fields that matter)

A `CaseStudy` object. Key fields, including the ones added during the Sfera job:

| field | notes |
|---|---|
| `slug` | URL segment. Kebab-case, descriptive. |
| `title` | Project name. |
| `status` | `"placeholder"` \| `"draft"` \| `"published"`. Use `"published"` for shipped, filled pages. Only `"placeholder"` shows a "Coming soon" chip. Status does **not** gate visibility. |
| `clientType` | e.g. `"Sfera 102.2 — music radio station"`. |
| `liveUrl` | **Added field.** Real outbound URL → renders a "Visit live site ↗" link in the Details meta block. Use this instead of putting `<a>` in body text. |
| `summary` | One-line lead under the title. |
| `problem` | Rendered as its own "Problem" section. |
| `role[]`, `services[]`, `tools[]` | String lists in the meta rail. |
| `tags[]` | **Max 3**, from the controlled taxonomy in [src/data/projectTags.ts](src/data/projectTags.ts). Pattern: one discipline + one method + one context (e.g. `["web", "wordpress", "culture"]`). |
| `cover` | `{ type, motif, alt, src?, full?, caption? }`. For a real image: `type: "image"`, `src: "media/work/<file>.webp"`, keep a `motif` as fallback, write a real `alt`. Add `full` (a full-page WebP) to make the cover **click-to-open the whole page** in a lightbox, and `caption` (e.g. `"Homepage"`) for the small label under it. |
| `gallery` | **Added field.** `Array<{ src, alt, full?, caption? }>`. Renders a 2-up grid of cropped tiles. **Landscape shots only** (see gotcha). `full` = full-page WebP opened in the lightbox on click; `caption` = small mono label under the tile (e.g. `"Production page"`). Omit `full` and the tile just opens `src`. |
| `sections[]` | `{ heading, body, mediaPlaceholder? }`. `mediaPlaceholder` is a dashed placeholder box — only for `[ADD SCREENSHOTS …]` text, **not** for real images (gallery handles those). |
| `outcomes[]` | Optional bullet list. |

## The pipeline

### 1. Read the live site (get real structure, not invented copy)

`WebFetch` was flaky on these sites (`ECONNRESET`). Prefer `curl` + parse:

```bash
curl -sL --max-time 25 -A "Mozilla/5.0 … Chrome/120 Safari/537.36" https://SITE/ -o site.html
```

Then pull `<title>`, meta, nav menu items (`li.menu-item > a`), and `h1/h2/h3`
headings to understand sections, and to confirm the stack (grep `wp-content|wp-json`
for WordPress). This gives the real IA to base copy on.

### 2. Screenshot with Playwright

- Playwright is **not** a project dependency and is **not** in `node_modules`. It lives in
  the **npx cache** under a hashed directory. **Never hardcode that path** — the hash
  (e.g. `e41f203b7505f1fb`) differs by machine, session, and agent, so a path copied from
  a previous run will break. **Always resolve it fresh**, then pass it in (the shoot
  scripts read it from `$PW_PATH`):
  ```bash
  PW_PATH=$(find ~/.npm/_npx -maxdepth 4 -name playwright -type d | head -1)
  echo "$PW_PATH"   # e.g. /Users/<you>/.npm/_npx/<hash>/node_modules/playwright
  ```
  If that returns nothing, prime the cache once with `npx playwright@latest --version`
  (or `npm i -D playwright` to make it a real dep), then re-run the `find`.
- It's a **CommonJS** module → `import pw from "$PW_PATH/index.js"; const { chromium } = pw;`
  (named `import { chromium }` fails).
- Chromium lives in the Playwright browser cache (`~/Library/Caches/ms-playwright/`, e.g.
  `chromium-1228/` + `chromium_headless_shell-1228/`). If it's missing: `npx playwright install chromium`.
- **Dismiss the cookie consent** or it blocks every shot. Greek IAB/Quantcast wall:
  the accept button is **"Συναίνεση"** (also try "Αποδοχή όλων", "Αποδοχή", "Accept all").
  Search **all frames** (`page.frames()`), retry a few times — it loads late.
- Capture the **homepage** for the cover, plus 2 clean **landscape** interior pages
  for the gallery (e.g. a chart/listing page and a schedule/detail page).
- **Also grab full-page shots** of every page you feature (cover + each gallery tile).
  These power the click-to-open lightbox (see §2b). So for each page you shoot a
  cropped viewport tile (`SLUG-home.png`) **and** a full-page version
  (`SLUG-home-full.png`).

Template script (`scratchpad/shoot.mjs` in the reference job):

```js
const pwPath = process.env.PW_PATH;          // path from the find above
const { chromium } = (await import(pwPath + "/index.js")).default;
const OUT = "public/media/work";             // absolute in practice

async function dismissCookies(page) {
  const labels = ["Συναίνεση","Αποδοχή όλων","Αποδοχή","Συμφωνώ","Accept all","Accept","AGREE","OK","Εντάξει"];
  for (let a = 0; a < 3; a++) {
    for (const frame of page.frames())
      for (const l of labels) {
        try {
          const b = frame.getByRole("button", { name: new RegExp("^\\s*"+l+"\\s*$","i") });
          if (await b.first().isVisible({ timeout: 500 })) { await b.first().click(); await page.waitForTimeout(800); return; }
        } catch {}
      }
    await page.waitForTimeout(700);
  }
}

async function shoot(ctx, url, file, { w=1440, h=900 } = {}) {
  const page = await ctx.newPage();
  await page.setViewportSize({ width: w, height: h });
  await page.goto(url, { waitUntil: "networkidle", timeout: 45000 }).catch(()=>{});
  await dismissCookies(page);
  await page.waitForTimeout(1500);
  await page.evaluate(() => window.scrollTo(0, 0));
  await page.screenshot({ path: `${OUT}/${file}` });   // viewport shot, not fullPage
  await page.close();
}

const browser = await chromium.launch();
const ctx = await browser.newContext({ deviceScaleFactor: 2 });
await shoot(ctx, "https://SITE/",        "SLUG-home.png");
await shoot(ctx, "https://SITE/page-a/", "SLUG-a.png");
await shoot(ctx, "https://SITE/page-b/", "SLUG-b.png");
await browser.close();
```

### 2b. Full-page shots for the lightbox

The cover and each gallery tile are **click-to-open**: clicking shows the whole page,
top-to-bottom, in a scrollable lightbox overlay (built into
[work/[slug].astro](src/pages/work/%5Bslug%5D.astro) — any case study gets it just by
adding `full` to `cover` / `gallery` items).

Capture these with `fullPage: true`, but **scroll the page first** so lazy imagery
loads, then scroll back to top:

```js
async function shootFull(ctx, url, file) {
  const page = await ctx.newPage();
  await page.setViewportSize({ width: 1440, height: 900 });
  await page.goto(url, { waitUntil: "networkidle", timeout: 45000 }).catch(()=>{});
  await dismissCookies(page);
  await page.evaluate(async () => {                 // trigger lazy content
    await new Promise((res) => {
      let y = 0;
      const step = () => { window.scrollBy(0, 600); y += 600;
        if (y < document.body.scrollHeight) setTimeout(step, 120); else res(); };
      step();
    });
  });
  await page.waitForTimeout(1200);
  await page.evaluate(() => window.scrollTo(0, 0));
  await page.screenshot({ path: `${OUT}/${file}`, fullPage: true });  // fullPage here
  await page.close();
}
await shootFull(ctx, "https://SITE/",        "SLUG-home-full.png");
await shootFull(ctx, "https://SITE/page-a/", "SLUG-a-full.png");
```

Full pages get **tall** (e.g. 2880×11870). That's fine — the lightbox scrolls.

### 3. Optimize images → WebP

- `sips` (built into macOS) resizes but **cannot output WebP** here. `cwebp` **is**
  installed (`/opt/homebrew/bin/cwebp`) — use it.
- Resize to 1600px wide (desktop), then encode WebP q80. Result ≈ 70–130 KB.
- The tall `-full` shots compress well too; use **q78** (≈ 200–360 KB) since they're
  only fetched when the lightbox opens.

```bash
cd public/media/work
# cropped viewport tiles (cover + gallery)
for f in SLUG-home SLUG-a SLUG-b; do sips --resampleWidth 1600 "$f.png" --out "$f.png" >/dev/null; done
for f in SLUG-home SLUG-a SLUG-b; do cwebp -q 80 -quiet "$f.png" -o "$f.webp" && rm "$f.png"; done
# full-page lightbox shots
for f in SLUG-home-full SLUG-a-full SLUG-b-full; do sips --resampleWidth 1600 "$f.png" --out "$f.png" >/dev/null; done
for f in SLUG-home-full SLUG-a-full SLUG-b-full; do cwebp -q 78 -quiet "$f.png" -o "$f.webp" && rm "$f.png"; done
```

### 4. Wire into `caseStudies.ts`

Set `cover.type: "image"` + `src` (+ `full` + `caption` for the click-to-open cover),
add `gallery` (landscape shots, each with `src` + `full` + `caption`), add `liveUrl`,
pick ≤3 tags, set `status`, and write the complete case-study copy. Use `[BRACKET]`
markers only for an unknown number, metric, date, or monetary amount. They render as
visible highlights via `withPlaceholders`.

### 5. Preview before declaring done

The site has `base: "/wireful"` ([astro.config.mjs](astro.config.mjs)) and a
`<base href="/wireful/">` tag, so image `src`s are **relative** (`media/work/…`, no
leading slash). To preview the built page you must serve it **under `/wireful`**:

```bash
npx astro build
SP=…/scratchpad
mkdir -p "$SP/serve"; ln -s "$PWD/dist" "$SP/serve/wireful"
cd "$SP/serve" && python3 -m http.server 8777 &
# open http://localhost:8777/wireful/work/SLUG/
```

Screenshot the rendered page with Playwright and **look at it**. Force reveal
animations first so content is visible in a static shot:
```js
await page.evaluate(() => document.querySelectorAll("[data-reveal],[data-reveal-group]").forEach(e => e.classList.add("is-visible")));
```
Check both the **detail page** and the **card** on `/wireful/work/`. Also **exercise
the lightbox**: click the cover and each gallery tile, confirm the full-page image
loads and scrolls, and that Escape / backdrop / Close all dismiss it. (Playwright note:
reveal transitions leave elements "not stable" — inject `*{transition:none}` and use
`{ force: true }` on hover/click.) Kill the server and remove the symlink when done.

## Gotchas / fixes already learned — don't repeat these

> **Keep this list growing.** Every time a new project throws something weird that
> needs a workaround — a different consent wall, a shell quirk, a broken crop, a stack
> that resists `curl`/Playwright — add it here (and bake reusable fixes into the shoot
> script) so the next project doesn't rediscover it.


1. **Image crop cut off headers.** Covers use `object-fit: cover` at 16/9. Default
   center position clipped the site logo/nav at the top. Fixed with
   `object-position: top center` on **both**:
   - `.case-cover__media img` (detail page — the cover box was restructured for the
     click-to-open lightbox + caption; the framed box is now `.case-cover__media`)
   - `.case-visual img` (listing card)
   in [src/styles/global.css](src/styles/global.css). Keep it; new covers rely on it.

2. **`withPlaceholders` escapes HTML.** Section `body` goes through
   [src/utils/placeholders.ts](src/utils/placeholders.ts), which HTML-escapes
   everything (so `[BRACKET]` tokens become visible marks). **Never put `<a>` or any
   markup in `body`** — it renders as literal text. For a live link use the `liveUrl`
   field instead.

3. **Gallery tiles are landscape.** They're cropped to 16/10 (`object-fit: cover`).
   A **portrait/mobile** screenshot cover-crops to an unreadable sliver — leave mobile
   shots out of `gallery`, or give them a dedicated letterboxed treatment first.

4. **Cookie consent blocks screenshots.** Must dismiss before shooting; the accept
   button label is Greek ("Συναίνεση") and often inside an iframe (search all frames).

5. **WebFetch is unreliable on these hosts** (`ECONNRESET`) — use `curl` to grab HTML.

6. **`sips` has no WebP** on this machine — use `cwebp`. `sips` is fine for resizing.

7. **Two shots per page: viewport tile + `fullPage`.** The cropped tile (cover/gallery)
   wants the hero/fold — a viewport shot. The lightbox wants the whole page — a
   `fullPage` shot (see §2b). Don't use a full-page grab as a tile: it's tall and messy.

8. **Write the copy, not the numbers.** Complete the narrative copy from the available
   evidence. Do not invent metrics, dates, counts, percentages, or monetary amounts.
   Keep `Result` qualitative unless a real number is supplied.

9. **Consent button label varies — "Accepting all", not "Accept all".** The Analysis
   site (analysis.ntlab.gr) uses a Quantcast/CMP wall whose accept control reads
   **"Accepting all"** (present continuous), so an exact-match `Accept all` regex
   misses it and the banner survives into every shot. Fixes now baked into the shoot
   script: put `Accepting all` in the label list, and add a `getByText` fallback next
   to `getByRole("button")` (some CMPs render the control as a link/div, not a button).
   When a fresh site keeps its banner, screenshot once, read the exact button text, and
   add it to `labels`.

10. **zsh does NOT word-split unquoted variables.** A loop like
    `tiles="a b c"; for f in $tiles` runs **once** with `f="a b c"` (fails), unlike
    bash. The playbook's `for f in SLUG-home SLUG-a SLUG-b` works only because the list
    is literal. If you factor filenames into a variable, use an array
    (`tiles=(a b c); for f in $tiles`) or `${=tiles}`. Simplest: keep the list literal.

11. **Lightbox `[hidden]` gotcha.** `.lightbox { display: flex }` and the `hidden`
   attribute's UA rule `[hidden] { display: none }` have **equal specificity**, so the
   later `.lightbox` rule wins and the overlay stays in the DOM at `opacity: 0`,
   silently covering the page and eating clicks. You **must** add an explicit
   `.lightbox[hidden] { display: none }` rule. Keep it.

12. **Some initiatives are genuinely single-page.** When a live site has anchored
   sections rather than interior URLs, capture clean landscape views of those sections
   for the gallery. Use the one full-page capture for their lightbox source, and name
   the captions after the featured sections.

13. **Playwright can be blocked by the execution sandbox on macOS.** A failed launch
    with `MachPortRendezvousServer ... Permission denied` is an environment permission
    issue, not a site or script issue. **Override:** rerun the browser command with
    the required elevated execution permission; do not keep changing the capture script
    or wait for a hung browser process.

14. **A static site may render incorrectly from its HTML alone.** REPower.eu uses
    relative CSS, font, image, and CSS-background asset paths. Saving only its HTML
    produced a washed-out, incomplete hero in local screenshots. **Override:** when
    the live host is slow or unreliable, mirror the referenced public assets into the
    same relative `css/`, `fonts/`, `images/`, and `js/` structure, then capture the
    local `file://` copy. Inspect the CSS for `url(...)` backgrounds as well as the
    HTML's `src`/`href` assets.

15. **`locator("#lightbox[hidden]").waitFor()` hangs by default.** Playwright waits
    for an element to become *visible* unless told otherwise; a correctly hidden
    lightbox is attached but not visible. **Override:** after Escape, backdrop, or
    Close, use `await page.locator("#lightbox[hidden]").waitFor({ state: "attached" })`
    (or assert the `hidden` attribute), rather than a default `waitFor()`.

16. **Video is supported on the case page (added for Pino).** For a heavily animated
    site, a static tile undersells it. The template now renders motion:
    - `cover.type: "video"` with `src` (mp4) + `poster` (webp) → the cover and the
      listing/carousel card both render a muted autoplay loop. `CaseCard.astro`,
      `work/[slug].astro`, and `.case-visual/.case-cover__media video` in `global.css`
      all handle it.
    - `gallery[]` items take an optional `type: "video"` + `poster`. Video tiles render
      inline (autoplay muted loop, a small "Motion" badge, `.case-shot__media`) and are
      **not** lightbox triggers. Static tiles keep the click-to-open button. Mixing both
      in one gallery is fine; the grid maps the whole array so >2 tiles works (Pino uses 6).
    - Keep the video count small (Pino: cover + 2 tiles). Posters + `preload="metadata"`
      keep it light; browsers only fetch/play video when it scrolls into view.

17. **Capturing GSAP/ScrollSmoother animations as video.** These sites (GSAP
    ScrollTrigger + ScrollSmoother, `#smooth-wrapper`/`#smooth-content`) can't be
    driven with `window.scrollTo`. Drive the real thing from `page.evaluate`:
    `ScrollSmoother.get().scrollTo(y, true)` (animated) or `scrollTo(el, true, "top top")`.
    Record with a Playwright context `recordVideo: { dir, size }` (one context per clip
    → one clean webm), then trim + encode with ffmpeg:
    `libx264 -profile:v high -crf 24 -preset veryslow -movflags +faststart -an`,
    `-vf scale=1280:-2,format=yuv420p`. Result ≈ 0.5–0.8 MB for ~4–8 s.
    - **Pinning shifts absolute offsets.** `getBoundingClientRect`-based section `y`s
      (from a probe) do not match real scroll positions once ScrollTrigger pins a section.
      Target the *element* (`scrollTo(el, ...)`) rather than a hardcoded `y`, or probe,
      capture, eyeball frames, and adjust.
    - **Playwright records ~0.5–1 s of black before first paint.** Trim the intro
      (`-ss ≈1.0`) so the clip and its poster start on content, not a blank frame.
    - **Poster must match the clip's resting/end state, not its start.** A poster pulled
      from the trim *start* can duplicate another tile (Pino's scatter clip started on the
      MANGIA/BEVI menu, doubling the Menu tile). Pull the poster from the frame that reads
      as the tile at rest (`ffmpeg -sseof -0.25 …` for the end frame).

18. **GitHub-Pages prototypes need no cookie wall** and serve over plain https, so the
    consent-dismissal dance (gotcha 4/9) is unnecessary. But do set the context
    `reducedMotion: "no-preference"` so the site's `prefers-reduced-motion` branch does
    not disable the animations you came to record.

## Quick checklist per project

- [ ] `curl` the site → confirm stack, read nav + headings
- [ ] Playwright: dismiss consent → shoot homepage + 2 landscape interiors (2× DSR)
- [ ] Also shoot `fullPage` versions of each featured page → `<slug>-*-full.png` (§2b)
- [ ] `sips` resize 1600w → `cwebp` (q80 tiles, q78 `-full`) → `public/media/work/<slug>-*.webp`
- [ ] Edit `caseStudies.ts`: `cover` (`src` + `full` + `caption`), `gallery` (same),
      `liveUrl`, ≤3 tags, `status`
- [ ] Copy: fill every narrative field; use `[BRACKET]` only for unverified numbers,
      metrics, dates, or monetary amounts
- [ ] `astro build`, serve under `/wireful`, screenshot + eyeball detail page **and** card
- [ ] Exercise the lightbox: cover + each tile open, scroll, and dismiss (Esc/backdrop/Close)
- [ ] Clean up temp server/symlink
